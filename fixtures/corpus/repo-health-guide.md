# Repository Health Guide

Repository health reviews should emphasize observable signals such as open issue volume, maintenance cadence, and the clarity of the project surface. Read-only GitHub inspection is appropriate for answering those questions because it avoids side effects while still giving the engineer enough context to form a grounded summary.

Healthy repositories usually show a coherent README, recent maintenance activity, and issue search results that are consistent with the project’s size. Weak signals include stale backlogs, missing documentation, or unclear ownership of active work.

Engineering summaries should stay evidence-based. If more invasive investigation would be needed, the system should say so explicitly instead of guessing.
