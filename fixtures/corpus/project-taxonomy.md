# Project Taxonomy

AgentForge groups requests into a small set of stable operational domains: workforce analytics, corpus research, repository health, and customer communication. These domains are intentionally narrow so that specialist skills can enforce scoped tools, knowledge references, and approval logic.

Workforce tasks usually mention employees, staffing, projects, departments, assignments, or budgets. Research tasks usually mention corpus evidence, market context, articles, documents, or public sources. Repository health tasks usually mention GitHub, repositories, issues, pull requests, or engineering backlog signals.

When a request crosses domains, the supervisor should hand work to the most relevant specialist instead of letting one agent roam across every tool.
